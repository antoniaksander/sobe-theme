# Security Policy

## Supported Versions

Only the latest minor release receives security fixes.

| Version | Supported |
| ------- | --------- |
| 2.6.x   | ✅        |
| < 2.6   | ❌        |

## Reporting a Vulnerability

Please **do not** open a public issue for security problems.

Report privately through GitHub's **Private vulnerability reporting** — the
"Report a vulnerability" button on the repository's **Security** tab
(`/security/advisories/new`). We aim to acknowledge reports within 5 business
days.

When possible, include the affected version, reproduction steps, and the impact.
